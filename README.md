# QtDataManager

Структура проекта реализует требования, описанные в `REQUIREMENTS.md`:

- архитектура MVC на базе Qt Widgets
- TreeView для дерева объектов -> передачи -> ТТХ
- TableView для редактирования ТТХ с делегатом
- подключение PostgreSQL через `DbManager`

## Сборка

```bash
mkdir build && cd build
cmake ..
cmake --build .
```

Убедитесь, что установлены Qt6 (Widgets, Sql) и доступна библиотека PostgreSQL (QPSQL).

Перед запуском выполните SQL-скрипт `scripts.sql` для создания схемы и тестовых данных:

```sh
psql -U user -d mydb -f ../scripts.sql
```

В `main.cpp` укажите параметры подключения (хост, порт, БД, пользователь, пароль).

## Файлы

* `dbmanager.*` — управление соединением с БД (поддерживает указание драйвера, по умолчанию QPSQL)
* `models/treemodel.*`, `treenode.*` — иерархическая модель
* `views/mainwindow.*` — основное окно с `QTreeView` и `QTableView`
* `delegates/valuedelegate.*` — делегат для редактирования значений

## Тесты

В проект добавлены модульные тесты с QtTest. После сборки их можно запустить через `ctest` или:

```sh
./test_treenode
./test_valuedelegate
./test_dbmanager
```

Тесты проверяют работу базовых компонентов без реальной БД (используется SQLite в памяти).
